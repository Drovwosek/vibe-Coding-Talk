Задача: исследуй ресторанное заведение «{name}» и подготовь рабочий профиль для маркетолога.

Подсказка для точного поиска: {hint}.

Что искать
- Официальный сайт или официальные страницы заведения.
- Карты и карточки заведения.
- Публикации самого заведения.
- Надёжные публичные упоминания.

Порядок работы
1. Найди источники, которые действительно относятся к указанному заведению.
2. Сравни найденные данные между источниками.
3. Заполни профиль только тем, что можно вывести из источников.
4. Раздели подтверждённые факты, осторожную маркетинговую интерпретацию и неопределённости.

Правила фактов
- `known_facts`: только сведения, подтверждённые источниками.
- `uncertainties`: всё неоднозначное, устаревшее, конфликтующее, неполное или требующее ручной проверки.
- Не придумывай адрес, кухню, средний чек, аудиторию, тон бренда, особенности, историю, акции или позиционирование.
- Если данные не найдены или сомнительны, прямо отметь это в `uncertainties`.
- Если есть несколько заведений с похожим названием, не смешивай их данные. Отметь риск путаницы в `uncertainties`.
- Не выполняй SWOT-анализ.
- Не выдумывай целевую аудиторию.

Как заполнять поля
- `name`: название заведения из наиболее надёжного источника.
- `description`: короткое описание только на основе найденных фактов.
- `positioning`: осторожная формулировка рыночной роли заведения. Если данных мало, напиши нейтрально и отметь ограничение в `uncertainties`.
- `voice`: предполагаемый тон коммуникации только по официальным текстам. Если официальных текстов мало, напиши нейтральную формулировку и отметь ограничение в `uncertainties`.
- `content_pillars`: 3-5 тем для контента, основанных на подтверждённых особенностях.
- `recommended_styles`: 3-5 коротких форматов Telegram-постов с пояснением подачи, соответствующих найденному бренду.

Формат ответа
- Верни только JSON по заданной схеме.
- Не добавляй markdown, комментарии или текст до/после JSON.
